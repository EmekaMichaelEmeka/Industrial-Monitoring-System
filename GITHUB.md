# GitHub upload guide

Use this checklist when publishing the project.

## Option A — Standalone repo (recommended)

Create a **new** GitHub repository (e.g. `industrial-monitoring-system`), then upload **everything inside** `industrial-monitor/` as the repo root.

## Option B — Subfolder in an existing repo

Upload only the `industrial-monitor/` folder into your existing repository (e.g. `abuad-src-complaints/industrial-monitor/`).

---

## Exact files to upload (17 files)

```
.gitignore
LICENSE
README.md
GITHUB.md
docker-compose.yml
start.ps1
start.sh
mqtt/mosquitto.conf
simulator/config.yaml
simulator/requirements.txt
simulator/sensor_simulator.py
simulator-cpp/CMakeLists.txt
simulator-cpp/src/main.cpp
dashboard/index.html
dashboard/styles.css
dashboard/app.js
nodered/flows.json
```

## Do NOT upload

| Path | Reason |
|------|--------|
| `simulator/.venv/` | Local Python environment — recreated with `pip install` |
| `simulator/__pycache__/` | Generated bytecode |
| `simulator-cpp/build/` | C++ build output — recreated with CMake |
| `.DS_Store`, `Thumbs.db` | OS junk |

Simulated pipeline sensors (pressure, temperature, flow, vibration),
Python/C++ MQTT publishers, Mosquitto broker, threshold alerts,
web dashboard, and optional Node-RED flow.

## Quick publish commands (standalone repo)

```bash
cd industrial-monitor
git init
git add .
git commit -m "Add industrial wellhead monitoring system with MQTT and live dashboard"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/industrial-monitoring-system.git
git push -u origin main
```

Replace `YOUR_USERNAME` and repo name with yours.
